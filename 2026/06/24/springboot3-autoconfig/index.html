Spring Boot 3 自动配置与起步依赖原理

一、什么是自动配置

Spring Boot 的核心思想是约定优于配置。自动配置（Auto-Configuration）会根据 classpath 中的 jar 依赖，自动完成 Spring 应用程序的配置，无需手动编写大量 XML 或注解。

二、@SpringBootApplication 的秘密

其中 @EnableAutoConfiguration 是自动配置的入口：

AutoConfigurationImportSelector 会读取 META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports 文件，加载所有自动配置类。

三、起步依赖（Starter）

Spring Boot 提供了一系列 Starter，简化依赖管理：

Starter 说明
spring-boot-starter-web Web 开发（内嵌 Tomcat + Spring MVC）
spring-boot-starter-data-jpa JPA + Hibernate
spring-boot-starter-data-redis Redis
spring-boot-starter-security Spring Security
spring-boot-starter-test 测试（JUnit 5 + Mockito）
spring-boot-starter-actuator 监控端点

四、条件注解（Conditional）

自动配置通过条件注解按需生效：

注解 条件
@ConditionalOnClass classpath 中存在指定类
@ConditionalOnMissingBean 容器中没有指定 Bean
@ConditionalOnProperty 配置文件中有指定属性
@ConditionalOnWebApplication 当前是 Web 应用
